<Heading level="1">
    miniMAL Architecture
</Heading>

<Paragraph>
    miniMAL is designed as a <Mark bold>Lisp-0</Mark> language where functions, symbols, and strings all share the same namespace. This makes it a very compact language while still being powerful.
</Paragraph>

<Heading level="2">
    Core Design Principles
</Heading>

<BulletedList>
    <BulletedList>
        <Mark bold>JSON Syntax</Mark>: All source code is valid JSON
    </BulletedList>
    <BulletedList>
        <Mark bold>Lisp-0 Model</Mark>: Single namespace for functions and symbols
    </BulletedList>
    <BulletedList>
        <Mark bold>Native Types</Mark>: Uses JavaScript native types (arrays for lists, objects for hash-maps)
    </BulletedList>
    <BulletedList>
        <Mark bold>TCO</Mark>: Automatic tail-call optimization
    </BulletedList>
</BulletedList>

<Heading level="2">
    Evaluation Model
</Heading>

<Paragraph>
    The core of miniMAL is the <Mark>EVAL</Mark> function which processes AST nodes:
</Paragraph>

<Paragraph blockColor="light_grey">
<Paragraph>
    <Mark>function EVAL(env, ast) {</Mark>
</Paragraph>
<Paragraph>
    <Mark>    // If not a list, evaluate atom</Mark>
</Paragraph>
<Paragraph>
    <Mark>    if (!Array.isArray(ast)) {</Mark>
</Paragraph>
<Paragraph>
    <Mark>        return eval_atom(ast, env);</Mark>
</Paragraph>
<Paragraph>
    <Mark>    }</Mark>
</Paragraph>
<Paragraph>
    <Mark>    // Process list forms</Mark>
</Paragraph>
<Paragraph>
    <Mark>    return apply_form(ast, env);</Mark>
</Paragraph>
<Paragraph>
    <Mark>}</Mark>
</Paragraph>
</Paragraph>
